# 手動E2E確認チェックリスト（音声タブコントローラー）

Playwright自動E2E（`e2e/voice-tab-controller.spec.ts`）は、開発環境の
Chromiumバイナリが起動できない問題（`side-by-side configuration
incorrect`、クリーン再インストール後も再発）により、この開発環境では
実行できていない。ユーザー承認のうえ、以下の手動確認へ切り替えている。
**実際にChromeが動く環境（リリース前）で、このチェックリストを
一度実行すること。**

## 準備

1. `npm run build`でビルドする
2. `chrome://extensions`を開き、デベロッパーモードを有効にする
3. 「パッケージ化されていない拡張機能を読み込む」で`dist/`を選択する

## シナリオ1: 基本表示・個別ミュート

- [ ] `e2e/fixtures/audio-page.html`をChromeで開き、「音声を再生」ボタンを押す
- [ ] 拡張機能アイコンをクリックし、popupに当該タブが「再生中」として表示される
- [ ] タブ一覧のミュートボタンを押し、実際に音が止まる（タブのスピーカーアイコンにも
      ミュート表示が出る）
- [ ] popupの状態表示が「ミュート（無音）」に変わる

## シナリオ2: 一括ミュート・一括解除

- [ ] 複数タブで音声を再生する
- [ ] 「一括ミュート」を押し、全対象タブが実際にミュートされ、成功件数が表示される
- [ ] 「一括解除」を押し、拡張機能がミュートしたタブが実際に解除される

## シナリオ3: ネイティブUI再ミュートの保護（自動テストで再現できない核心シナリオ）

- [ ] 音声再生タブを一括ミュートする
- [ ] Chromeのタブのスピーカーアイコンを**直接クリック**してネイティブUIで解除する
- [ ] 同じくネイティブUIのスピーカーアイコンで**再度ミュート**する
      （これで`mutedInfo.reason`が`'user'`になる）
- [ ] popupの「一括解除」を押しても、**このタブは解除されない**ことを確認する
      （凍結設計で修正した安全性ガードの核心。ここが壊れていたら重大なバグ）

## シナリオ4: 常時ミュート

- [ ] popupで現在のサイトを常時ミュートに登録する
- [ ] 同じサイトの新規タブを開き、自動的にミュートされることを確認する
- [ ] 常時ミュート対象タブの個別ミュートボタンが無効化され、理由がtooltip等で
      表示されることを確認する
- [ ] 常時ミュート対象タブをChromeのタブスピーカーアイコンで手動解除しても、
      background側が自動的に再ミュートすることを確認する（実装レビューで
      発見・修正したmajor：popupのボタン無効化だけではネイティブUIからの
      解除を防げない）
- [ ] 常時ミュート規則を削除しても、既存タブが自動的に解除されないことを確認する

## シナリオ5: Service Worker再起動後の永続性

1. [ ] `chrome://extensions`の拡張機能詳細から「Service Worker」インスペクタを開く
2. [ ] 常時ミュート規則を登録する
3. [ ] インスペクタを閉じ、Service Workerが`inactive`になるまで待つ
      （`chrome://extensions`の表示で確認）
4. [ ] 登録済みサイトへ新規タブで遷移し、Service Workerが再起動して自動ミュートが
      適用されることを確認する
5. [ ] 規則削除後、同様に再起動を挟んでも自動ミュートされないことを確認する

## シナリオ6: 一括ミュート後、Service Worker再起動を挟んでの一括解除（blocker修正の実機確認）

実装レビューで発見された最重要のblocker——「一括ミュートした対象を、
Service Worker再起動後に一括解除しようとすると機能しなくなる」——の
実機確認。`chrome.storage.session`への切り替えで修正済みだが、実機での
確認が必須。

1. [ ] 音声タブを一括ミュートする
2. [ ] popupを閉じ、Service Workerが`inactive`になるまで待つ
      （`chrome://extensions`のService Workerインスペクタで確認）
3. [ ] 再度popupを開き、「一括解除」を押す
4. [ ] 一括ミュートしたタブが正しく解除されることを確認する
      （**ここが機能しない場合、blockerが再発している**）
